# GitHub Workflow Standards

## Branch Naming
Format: `[type]/[short-description]`
- `feat/` — New feature
- `fix/` — Bug fix
- `refactor/` — Refactoring
- `docs/` — Documentation

## Commit Messages
Format: `[type]: [description]`
Keep under 72 chars. Imperative mood.

## PR Standards
- Descriptive title
- Body explaining what/why
- Labels for tracking
- Linked issues if applicable

## Merge Strategy
Default: Squash merge (clean history)
